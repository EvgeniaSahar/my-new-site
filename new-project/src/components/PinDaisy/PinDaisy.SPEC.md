# PinDaisy — спецификация для разработчика

Демо: https://evgeniasahar.github.io/my-new-site/pins/

## 1. Назначение

`PinDaisy` — интерактивная «ромашка» из 1–28 пинов с ценами. Пользователь двигает ползунок количества пинов и переключает режим отступов **compact** / **spacious**. Каждый пин позиционируется вручную по референсам; горизонтальные расстояния считаются по реальной ширине DOM-элементов.

**Файлы:**

| Файл | Роль |
|------|------|
| `PinDaisy.jsx` | Логика, измерения, рендер |
| `layouts.js` | Базовые позиции для count 1…28 |
| `PinDaisy.css` | Стили |
| `PinDaisy.SPEC.md` | Эта спецификация |

---

## 2. Координатная система

| Параметр | Значение |
|----------|----------|
| Логический бокс | 220×220 px, центр в (0, 0) |
| Viewport по умолчанию | 260×260 px (настраивается ползунками) |
| Высота пина | 28 px → половина `PIN_HALF_H = 14` |
| Ось Y | вниз положительная |
| Позиционирование | `transform: translate(-50%, -50%) translate(x, y)` |

Пин центрируется по своей точке `(x, y)`.

---

## 3. Режимы отступов

В коде: `spacingType` — `1` (compact) или `2` (spacious). В UI: кнопки **compact** / **spacious**.

### 3.1. Compact (`spacingType === 1`)

Базовые значения из `layouts.js` + точечные override в `useMemo`.

| Группа | Gap |
|--------|-----|
| Центральная колонка 1–2–3, 18–19 | 4 px |
| Правые пары 4–5, 6–7, 14–15, 16–17, 24–25, 26–27 | 8 px |
| Горизонтальные ряды 8–9, 10–11, 12–13 | 68 px |
| Горизонтальные ряды 20–21, 22–23 | 48 px |
| Пин 28 | `y = 92` (layout 88 + 4) |

### 3.2. Spacious (`spacingType === 2`)

Только отличия от compact:

| Группа | Compact | Spacious |
|--------|---------|----------|
| Центр: 1–2–3, 18–19 (вертикаль) | 4 px | **6 px** |
| Правая колонка: 4, 6, 14, 16, 24, 26 (вертикаль) | 4 px | **6 px** |
| Правая inset-колонка: 8, 10, 12, 20, 22 (вертикаль) | 4 px | **6 px** |
| Пары 4–5, 6–7, 14–15, 16–17, 24–25, 26–27 (горизонталь) | 8 px | **12 px** |
| Ряды 8–9, 10–11, 12–13 (горизонталь) | 68 px | **80 px** |
| Ряды 20–21, 22–23 (горизонталь) | 48 px | **60 px** |
| Пин 28 | `y = 92` | **`y = 98`** |

## 4. Модель данных позиции (`layouts.js`)

Каждый пин — объект с полями-примитивами:

```js
{ x: 0, y: 0 }                          // абсолютная позиция от центра
{ xPairGap: 8, y: -16, mirror: true }   // горизонтальная пара от центра
{ xHalfPin: true, xHalfGap: 4, y: 16 }  // смещение на полширины + gap
{ xRightInset: 8, y: 0, z: 30 }         // отступ от правого края бокса
{ xGapFrom: 7, gap: 68, y: 0 }          // слева от другого пина с gap
{ z: 50 }                               // явный z-index
```

`PIN_LAYOUTS[count]` — массив из `count` позиций. Индекс 0-based: пин N → индекс `N - 1`.

---

## 5. Пайплайн расчёта позиций

```
PIN_LAYOUTS[count]
       ↓
positions (useMemo)     ← Y-override по spacingType
       ↓
useLayoutEffect         ← измерение DOM, расчёт X
       ↓
render                  ← финальные (x, y), z-index
```

### 5.1. `positions` (useMemo)

Берёт layout для текущего `count`, применяет Y-коррекции.

**Spacious — центральная колонка:**

| Пин | Compact y | Spacious y |
|-----|-----------|------------|
| 2 | -32 | -34 |
| 3 | 32 | 34 |
| 18 | -64 | -68 |
| 19 | 64 | 68 |

**Spacious — правая колонка** (`RIGHT_COL_STACK_INDEXES`):

Якорь — пин 4 (`y = -16`). Шаг `TYPE2_RIGHT_COL_STEP = 34` (14 + 6 + 14).

| Пин | Compact y | Spacious y |
|-----|-----------|------------|
| 4 | -16 | -16 |
| 6 | 16 | 18 |
| 14 | -48 | -50 |
| 16 | 48 | 52 |
| 24 | -80 | -84 |
| 26 | 80 | 86 |

**Spacious — правая inset-колонка** (`RIGHT_INSET_STACK_INDEXES`):

Якорь — пин 8 (`y = 0`). Шаг `TYPE2_EDGE_Y = 34`.

| Пин | Compact y | Spacious y |
|-----|-----------|------------|
| 10 | -32 | -34 |
| 8 | 0 | 0 |
| 12 | 32 | 34 |
| 20 | -64 | -68 |
| 22 | 64 | 68 |

**Compact-only Y:**

- Пин 14 над пином 4: `TYPE1_PIN14_ABOVE_PIN4_Y = -48`
- Пин 16 под пином 6: `TYPE1_PIN16_BELOW_PIN6_Y = 48`

**Пин 28:** compact `y = 92`, spacious `y = 98`.

### 5.2. `useLayoutEffect` — расчёт X

После рендера читаются `offsetWidth` пинов и считаются смещения.

**Формула для горизонтальной пары** (edge gap между краями = `G`):

```
offset = G/2 + (widthA + widthB) / 4
```

Если второго пина ряда ещё нет (ползунок на N, пара появляется при N+1):

```
offset = G/2 + widthA/2
```

Правый пин ряда: `+offset`, левый: `-offset`.

> **Важно:** не ставить `x = 0` и не ждать появления соседа — иначе пин прячется под центральным пином с высоким z-index.

**Типы горизонтального X:**

| Тип layout | Логика |
|------------|--------|
| `xPairGap` | `±pairOffset` или `±pairOffset45` (пины 4–5) |
| `xHalfPin` | `±halfPinOffset67` (пины 6–7) или `halfPinW + xHalfGap` |
| `xRightInset` | `LAYOUT_BOX_W/2 - inset - width/2` |
| `xGapFrom` | позиция соседа − gap − полуширины |
| Центрированный ряд | `±pin89Offset`, `±pin1011Offset`, и т.д. |

`resolveGap(i, p, spacingType)` подменяет gap из layout на актуальный для spacious (68→80, 48→60, 8→12).

### 5.3. Render — выбор X

Цепочка `if/else` в `PinDaisy.jsx` (блок `positions.map`). Приоритет:

1. Центральная колонка (`x === 0`) → `x = 0`
2. Центрированные ряды 8–9, 10–11, 12–13, 20–21, 22–23
3. Специальные ряды 14, 16, 24, 26
4. `gapFromXs`, `rightInsetXs`, `xPairGap`, `xHalfPin`, `p.x`

**Z-index:** активный пин 1500; центр 2000; остальные по расстоянию от центра или явный `p.z`.

## 7. Функции-детекторы рядов

```js
isPin89Row / isPin1011Row / isPin1213Row   // xRightInset + xGapFrom, фиксированный rowY
isPin2021Row / isPin2223Row               // то же, rowY −64 / 64
isPin1617Row                              // xHalfPin + xGapFrom
isPin2425Row / isPin2627Row               // xPairGap + xGapFrom
isPin14Type1AbovePin4                     // пин 14 над пином 4 (compact)
```

`isCenteredPinRow` сравнивает `p.y` с **уже преобразованным** Y (через `insetRowY` для spacious). Иначе ряд не распознается после Y-override.

## 9. Типичные баги и как их избегать

### Пин не виден при count = N, виден при N+1

**Причина:** пин ряда получает `x = 0` и перекрывается центральным пином (`z: 2000`).

**Фикс:**

1. При одном пине ряда: `offset = gap/2 + width/2`
2. В `resolvedXs` и render — всегда применять offset, без `count > PIN(N+1)_INDEX`
3. Не привязывать offset к ref соседа (ref[N] — это пин N+1, не N)

Затронутые ряды: 8, 10, 12, 16, 20, 22, 24, 26.

### Spacious: ряд не попадает в `isPinXXRow`

**Причина:** `p.y` уже сдвинут (например −34), а детектор сравнивает с layout Y (−32).

**Фикс:** `insetRowY(rowY, spacingType)` в `isCenteredPinRow`.

### Неверный горизонтальный gap в spacious

**Причина:** в layout жёстко `gap: 8`, а spacious нужен 12.

**Фикс:** `resolveGap()` + константы `TYPE2_*_EDGE_GAP`.

## 10. Схема колонок

```
        [18]
        [10]     [20]
        [8]      [9]
[14]    [1]      [12]    [13]
[4][5]  [2]      [11]
[6][7]  [3]      [22]    [23]
[16]    [19]     [21]
[24][25]
[26][27]
              [28]
```

Вертикальные стеки:

- **Центр (x = 0):** 18–10–8–12–22 (inset) и 1–2–3, 18–19
- **Правая колонка (xPairGap / xHalfPin):** 14–4–6–16–24–26
- **Горизонтальные пары** отходят влево от правых пинов рядов

## 12. Чеклист при добавлении нового пина

1. Добавить позицию в `PIN_LAYOUTS` для нужных count (и всех последующих).
2. Определить тип позиционирования (`xPairGap`, `xRightInset`, …).
3. Если ряд центрированный — добавить детектор и offset в `useLayoutEffect` + render.
4. Проверить ползунок на count, когда пин один в ряду, и на count+1.
5. Проверить оба режима compact и spacious.
6. Пересобрать и задеплоить на `gh-pages`.
